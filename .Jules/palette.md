## 2026-01-24 - Broken Promises in Onboarding
**Learning:** A non-functional "Skip" button in onboarding is worse than no button at all. It signals to the user that the app is buggy or unfinished before they even start using it.
**Action:** When implementing onboarding screens, treat navigation buttons as "critical path" features that must be implemented immediately, not left as TODOs.
